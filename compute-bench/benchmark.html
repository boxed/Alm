The benchmark above is DOM-bound — every operation crosses into the browser, which hides the compute backend's real speed. These are pure-computation workloads (no DOM, no I/O) running the same source through alm's three backends: compiled to JavaScript, to WebAssembly-GC, and to a native binary. This is where WasmGC's advantage shows: it beats the JS backend on every workload — up to 3.4× on the map/filter/fold pipeline, now that wasm-gc fuses those combinators against a known callee (a specialized per-call loop that inlines the lambda body, dropping the per-element apply1 dispatch), 1.5–1.7× on dict/sort, and, after unboxing numbers and inlining numeric constants, ~2× on recursion and ~1.05× on float math too.

How the DOM benchmark was run. krausest/js-framework-benchmark, keyed implementations, puppeteer driving system Google Chrome headless. elm, alm and alm-wasm share Main.elm verbatim: elm via the official compiler, alm to JS (both minified with the same uglify pass), alm-wasm to a WebAssembly-GC module driven by a hand-written JS shim (DOM host imports; no WASI). Duration is the median of 15 timed iterations; memory is a single sample.

How the computation benchmark was run. Each workload is a value program (main : Int) compiled by alm to three targets from identical source. JS: the exported function called in-process under Node/v8. wasm-gc: the module's entry called on the instantiated module. native: the unboxed --target=native-typed binary. All three are checked to return the same result before timing; time is the median of 25 timed calls after 10 warmup. The JS and wasm figures are timed warm and in-process; the native figure is the min of 12 process runs and so carries a fixed ~2.6 ms of process startup (exec + heap init) that the others don't — an asymmetry that weighs on the smaller native workloads. Native uses a custom Immix-lite garbage collector (bump allocation + conservative mark-region reclaim), now the default — it replaced Boehm entirely, cutting binary-trees from ~16.5 to 10 ms, Dict ~43 to 33, sort ~17 to 12.6. It's fastest on scalar loops and competitive elsewhere; the one place it still trails wasm is the weight-balanced Dict tree's pointer-chasing versus v8's JIT + generational GC.
